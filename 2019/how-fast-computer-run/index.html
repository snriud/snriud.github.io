计算机中的各部件运行有多快？

计算机中各部件的典型速度

计算机中各部件有不同的执行速度，每一个程序员都应该了解，并且能有效地利用好它们。下面的一张图中，列出了各主要部件的典型速度，他们所处的不同的量级，应该做到心中有数。

图中不仅列出了时间，还列出了一个放大后的时间，我们可以很容易地映射到我们生活中常用的时间尺度，对于他们之间的速度差异有一个更加直观的感受。

了解了这些，在平时写代码的时候，就要去考虑，如何能够让程序更加高效地运行。CPU 为什么要有流水线，为什么要设置那么多层级的缓存，数据库系统中为什么要利用 redo log？为什么在循环中进行IO操作甚至网络操作的行为是愚蠢的？

glibc 中 IO 相关函数

CPU 的运算速度要比 IO 操作执行得快得多。因此，在读取或写入数据到磁盘时，glibc 中的相关函数在系统调用之上，用户空间之中又增加了一层缓存/缓冲，使得一次系统调用能够操作较多的数据，另外，在 CPU 遇到需要等待 IO 操作时，切换至其它进程运行，而不是等待。

服务端程序业务逻辑中的运用

想像一种业务逻辑要求：获取一个用户集合中每个用户的订单信息。新手很可能会在遍历用户集合的过程中不断进行数据库访问，获取该用户的订单集合。数据库访问一般来说是网络请求，如果了解到网络请求和 CPU 运算速度之间的巨大差别，是不可能这样去实现的。
